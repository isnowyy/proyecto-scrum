# Definición de Hecho (Definition of Done)

## Proyecto
Sistema de Gestión de Tareas

## Descripción
La **Definición de Hecho** establece los criterios que deben cumplirse para considerar que una funcionalidad o incremento del producto está completamente terminado. Esto permite mantener la calidad del proyecto y asegurar que todas las tareas cumplan con los estándares definidos por el equipo.

---

## Criterios generales para considerar una tarea como "Hecha"

Una tarea se considera completada cuando cumple con los siguientes puntos:

- La funcionalidad fue desarrollada según los requerimientos.
- El código funciona correctamente sin errores.
- El código fue probado por al menos un integrante del equipo.
- La funcionalidad fue integrada al repositorio del proyecto en GitHub.
- La documentación del proyecto fue actualizada.
- El equipo revisó y aceptó la funcionalidad desarrollada.

---

# Criterios de aceptación de los incrementos

Los criterios de aceptación permiten verificar que las funcionalidades desarrolladas cumplen con los requisitos definidos en las historias de usuario.

---

## Historia de usuario 1
**Como usuario quiero crear tareas para poder organizar mis actividades.**

### Criterios de aceptación
- El sistema permite ingresar el nombre de una tarea.
- La tarea se guarda correctamente en el sistema.
- El sistema muestra un mensaje de confirmación al crear la tarea.
- La tarea queda registrada en el archivo JSON.

---

## Historia de usuario 2
**Como usuario quiero listar mis tareas para ver las actividades pendientes.**

### Criterios de aceptación
- El sistema muestra todas las tareas registradas.
- Cada tarea debe mostrar su nombre.
- El sistema debe mostrar las tareas de forma ordenada.

---

## Historia de usuario 3
**Como usuario quiero marcar tareas como completadas para saber cuáles ya terminé.**

### Criterios de aceptación
- El sistema permite cambiar el estado de una tarea a completada.
- El estado de la tarea debe actualizarse correctamente.
- La tarea completada debe mostrarse como finalizada en la lista.

---

## Conclusión
La definición de hecho y los criterios de aceptación permiten garantizar que cada incremento del producto cumpla con los estándares de calidad definidos por el equipo y que las funcionalidades entregadas sean útiles para el usuario final.
