# Evidencia de Refinamiento del Backlog

## Fecha
13 de marzo de 2026

## Participantes
- Product Owner
- Scrum Master
- Equipo de desarrollo

## Actividades realizadas

Durante la sesión de refinamiento del backlog el equipo revisó las historias de usuario del proyecto y realizó mejoras en su definición.

Se realizaron las siguientes actividades:

- Se revisaron las funcionalidades principales del sistema.
- Se aclararon dudas sobre el funcionamiento del sistema.
- Se trato de trabajar con manejo de errores para un proyecto mas limpio y sin errores


## Cambios realizados en el Product Backlog

| Historia de usuario | Cambio realizado |
|---|---|
| Crear tareas | Se agregó validación de datos |
| Listar tareas | Se definió formato de visualización |
| Guardar tareas | Se especificó el uso de archivo JSON |

## Conclusión

El refinamiento permitió que el equipo tuviera una mejor comprensión de las tareas del proyecto y facilitó la planificación del sprint.
