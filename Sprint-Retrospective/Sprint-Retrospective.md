# Sprint Retrospective

## Proyecto
Sistema de Gestión de Tareas

## Sprint
sprint final

## Fecha
12 de marzo de 2026

## Participantes
- Scrum Master
- Product Owner
- Equipo de desarrollo

---

## Objetivo de la retrospectiva
Analizar cómo trabajó el equipo durante el sprint, identificar los aspectos positivos, los problemas encontrados y definir acciones de mejora si es necesario.

---

## Lo que salió bien

Durante el sprint el equipo logró organizarse correctamente y mantener una buena comunicación. Se realizaron las reuniones necesarias para revisar el progreso del proyecto y se logró completar las funcionalidades principales planificadas para este sprint.

Aspectos positivos:
- Buena colaboración entre los integrantes del equipo.
- Se cumplieron las tareas principales del Sprint Backlog.
- Se logró implementar correctamente las funciones para crear y listar tareas.
- El equipo utilizó GitHub para compartir y actualizar el código.

---

## Lo que no salió tan bien

A pesar de los avances logrados, también se identificaron algunos problemas que afectaron el desarrollo del proyecto.

Dificultades encontradas:
- Algunos integrantes tuvieron dificultades al trabajar con Git y las ramas del repositorio.
- Hubo pequeños retrasos en la asignación de tareas.
- En algunos momentos faltó comunicación sobre el estado de ciertas actividades.
- se hizo un commit sin querer del desarrollador 3 que tenia unos archivos que no pertenecian a la rama del desarrollador y hizo push sin darse cuenta, pero se arreglo borrando el archivo y haciendo commit con todo actualizado.

---

## Qué podemos mejorar

Después de analizar el sprint, el equipo propuso algunas mejoras para optimizar el trabajo en el próximo sprint.

Mejoras propuestas:
- Practicar más el uso de Git y el manejo de ramas.
- Organizar mejor las tareas en el tablero Kanban.
- Mantener una comunicación constante durante el desarrollo del sprint.
- Realizar el Daily Scrum todos los días para revisar el progreso.

---

## Acciones para el próximo Sprint

Para mejorar el rendimiento del equipo en el siguiente sprint se acordaron las siguientes acciones:

- Revisar el código y documentacion antes de subirlo al repositorio.
- Actualizar el tablero de tareas diariamente.
- Asignar responsabilidades claras a cada integrante.
- Realizar seguimiento constante a las tareas pendientes.

---

## Conclusión

El equipo considera que el sprint fue productivo, ya que se lograron desarrollar varias funcionalidades importantes del sistema. Sin embargo, también se identificaron oportunidades de mejora relacionadas con la organización del trabajo y el uso de herramientas de control de versiones. Estas mejoras serán aplicadas en el siguiente sprint para lograr un mejor rendimiento del equipo.
